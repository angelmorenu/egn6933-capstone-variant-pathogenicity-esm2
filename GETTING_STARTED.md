## Understanding Project Scope

### Individual Project
- **Recommended approach** by the instructor
- Full autonomy and ownership
- Manageable scope for one person
- Clear individual contribution

### Group Project (2 people)
- **Requires approval** from faculty advisor
- Must have expanded scope to justify team size
- Need clear rationale for why group work is necessary
- Both members must have distinct, substantial contributions
- Consider coordination overhead

**Note:** Larger groups may be considered with approval

## Tips for Success

### Project Selection
1. **Be Realistic:** Can you complete this in one semester?
2. **Data First:** Ensure you can access the data you need
3. **Iterative Approach:** Plan for experimentation and refinement
4. **Define Success:** What does a successful project look like?

### Working with Your Advisor
1. **Regular Communication:** Schedule consistent check-ins
2. **Come Prepared:** Have specific questions and updates
3. **Be Receptive:** Accept feedback gracefully
4. **Document Everything:** Keep meeting notes

### Project Management
1. **Start Early:** Don't procrastinate on data collection
2. **Version Control:** Use Git for your code
3. **Document as You Go:** Don't save documentation for the end
4. **Plan for Setbacks:** Things will take longer than expected

### Technical Best Practices
1. **Reproducibility:** Make your work reproducible
2. **Code Quality:** Write clean, well-documented code
3. **Experiments:** Track all experiments and results
4. **Backup:** Keep backups of your data and code
